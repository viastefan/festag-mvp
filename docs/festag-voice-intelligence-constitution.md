## CHAPTER 1: THE VISION / DIE VISION

### English

Festag is not building a voice assistant. Festag is building an Operational
Intelligence System. Voice is only the interface through which organizational
knowledge, decisions and execution flow. Every interaction should reduce
complexity, create clarity, capture decisions and improve the company's
knowledge model.

### Deutsch

Festag entwickelt keinen Sprachassistenten, sondern ein
Operational-Intelligence-System. Sprache dient lediglich als natürliches
Interface. Jede Unterhaltung soll Wissen erfassen, Entscheidungen strukturieren,
Klarheit schaffen und das Unternehmensmodell verbessern.

### Implementation Principles

1. Prefer conversations over dashboards.
2. Never ask for information already known.
3. Every conversation must produce structured knowledge.
4. Every decision updates projects, tasks and priorities.
5. Optimize for long-term organizational intelligence.

### Anti-Patterns

- Do not build generic chatbot experiences.
- Do not overwhelm users with updates.
- Do not lose conversational context.
- Do not require manual synchronization.

### Cursor Rules

Persist these principles across future implementations. Treat all future
chapters as cumulative. Never contradict previous chapters.

## Merge map (v1.0 ↔ existing doctrine)

| Voice Constitution | Existing doctrine |
|---|---|
| Voice = interface, not product | Adaptive Intelligence: Organizational Intelligence is the product |
| Prefer conversations over dashboards | North star: calm clarity surfaces; executive/management by exception — conversation complements, does not delete role portals |
| Never ask for known information | Tagro: search company knowledge / OKM first |
| Conversation → structured knowledge | OKM facts, decisions, evidence, work signals |
| Decisions update projects/tasks/priorities | Decision orchestration + objectives/tasks linkage |
| No generic chatbot | Tagro/Veyra as interpreters |
| No manual sync | Connector / signal pipeline; write-back to project state |
| Long-term organizational intelligence | Continuous improvement loop + Company Brain |

### Non-contradiction notes

- **“Prefer conversations over dashboards”** does not abolish `/executive`,
  Client Panel, or Evidence surfaces. It means conversation is the primary
  *interaction mode* for capture and steering; dashboards remain *truth
  displays* for the right role.
- Voice/audio briefings already in product docs remain **outputs** of
  intelligence (always with transcript), not a separate “voice product.”
- Privacy rules from Adaptive Intelligence still bind: collaboration
  intelligence, workspace scope, personal profiles opt-in.
